# API Documentation

Full interactive documentation (with live request/response schemas and a "try it out"
console) is auto-generated by FastAPI and served at **`/docs`** (Swagger UI) and
**`/redoc`** (ReDoc) once the backend is running. This file is a static reference summary.

**Base URL:** `http://localhost:8000/api/v1` (development)
**Auth:** Bearer JWT in the `Authorization` header, obtained from `/auth/login`.

---

## Authentication (`/auth`)

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/auth/register` | none | Create a new account. `201` → `UserResponse` |
| POST | `/auth/login` | none | OAuth2 password flow (form-encoded `username`/`password`). `200` → `Token` |
| POST | `/auth/refresh` | none (refresh token in body) | Exchange a refresh token for a new token pair. `200` → `Token` |
| GET | `/auth/me` | required | Current user's profile. `200` → `UserResponse` |
| POST | `/auth/verify-email?token=...` | none | Confirm email via the token sent at registration |
| POST | `/auth/change-password` | required | `204` on success |
| POST | `/auth/forgot-password` | none | Always `202`, regardless of whether the email exists (prevents enumeration) |
| POST | `/auth/reset-password` | none (reset token in body) | `204` on success |

**Example — login:**
```
POST /api/v1/auth/login
Content-Type: application/x-www-form-urlencoded

username=asha.rao%40crss.edu&password=Password123!
```
```json
200 OK
{
  "access_token": "eyJhbGciOi...",
  "refresh_token": "eyJhbGciOi...",
  "token_type": "bearer"
}
```

## Resources (`/resources`)

| Method | Path | Auth | Description |
|---|---|---|---|
| GET | `/resources` | required | Search/filter/sort/paginate. Query params: `search`, `category_id`, `condition`, `status`, `department`, `min_rating`, `sort_by`, `sort_dir`, `page`, `page_size` |
| GET | `/resources/{id}` | required | Detail (also increments `view_count`) |
| POST | `/resources` | required | Create a listing |
| PUT | `/resources/{id}` | owner or admin | Partial update |
| DELETE | `/resources/{id}` | owner or admin | `204` on success |
| POST | `/resources/{id}/images` | owner or admin | Attach an already-hosted image URL |

**Example — search:**
```
GET /api/v1/resources?search=camera&condition=good&sort_by=average_rating&sort_dir=desc&page=1&page_size=12
```
```json
200 OK
{
  "total": 1,
  "page": 1,
  "page_size": 12,
  "items": [ { "id": "...", "title": "Canon EOS 1500D DSLR Camera", "...": "..." } ]
}
```

## Borrow Requests (`/borrow-requests`)

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/borrow-requests` | required | Create a request (`400` if self-borrow or out of stock) |
| GET | `/borrow-requests/my-requests` | required | Requests you've made as a borrower |
| GET | `/borrow-requests/incoming` | required | Requests on your resources, as a lender |
| POST | `/borrow-requests/{id}/approve` | lender or admin | `400` if not currently `requested` |
| POST | `/borrow-requests/{id}/reject` | lender or admin | Body: `{"rejection_reason": "..."}` |
| POST | `/borrow-requests/{id}/cancel` | borrower | Only while `requested` or `approved` |
| POST | `/borrow-requests/{id}/return` | borrower | Body: `{"damage_report": "..."}` (optional) |

**State machine:** `requested → approved → returned` (or `damaged`), with `rejected` /
`cancelled` as terminal off-ramps from `requested`/`approved`.

## Reviews & Notifications

| Method | Path | Auth | Description |
|---|---|---|---|
| POST | `/reviews` | required | `400` if the reviewer has no `returned` borrow for that resource |
| GET | `/resources/{id}/reviews` | required | All reviews for a resource |
| GET | `/notifications` | required | Current user's notifications, newest first |
| POST | `/notifications/{id}/read` | required | Mark one as read |
| POST | `/notifications/read-all` | required | Mark all as read |

## WebSocket

`ws://<host>/ws/notifications?token=<jwt>` — real-time push of the same payload shape
returned by `GET /notifications`, delivered the instant a notification is created
server-side (falls back gracefully to polling `/notifications` if the socket drops).

## Standard Error Shape

Every error response (validation, not-found, forbidden, conflict, server error) returns:
```json
{
  "detail": "Human-readable message (or a list of field errors for 422s)",
  "error_code": "MACHINE_READABLE_CODE"
}
```

| HTTP Status | Typical `error_code` values |
|---|---|
| 400 | `SELF_BORROW`, `OUT_OF_STOCK`, `INVALID_STATE`, `NOT_ELIGIBLE`, `BAD_PASSWORD`, `INVALID_FILE_TYPE`, `FILE_TOO_LARGE` |
| 401 | `BAD_CREDENTIALS`, `BAD_REFRESH_TOKEN` |
| 403 | `FORBIDDEN`, `ACCOUNT_DISABLED`, `CSRF_FAILURE` |
| 404 | `NOT_FOUND`, `USER_NOT_FOUND` |
| 409 | `CONFLICT`, `DB_INTEGRITY` |
| 422 | `VALIDATION_ERROR` |
| 429 | (slowapi's standard rate-limit response) |
| 500 | `DB_ERROR`, `INTERNAL_ERROR` |
